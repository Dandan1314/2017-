1.基本数值类型保存的是真实的值,而引用类型保存的是存在内存的地址 基本数据类型复制变量值非常简单,只需要num1=6 ; num2 = num1; 就可以了,并且他们是两个独立的值; 引用类型,复制是复制对象的指针,他们都指向同一个地址 2.检测类型 基本数据类型 使用typeof; 引用类型,可以使用instanceof来判断是哪一种对象 注意: 在ie和火狐浏览器中,使用typeof检测function会放回objert; 其他浏览器返回function 3.延长作用域链; try catch语句 with语句 注意:ie9之前的浏览器,在catch语句外也能访问到内部的变量 4,没有块级作用域 根据标识符来获取变量,直到window(全局环境) 5.垃圾收集 javaScript具有自动垃圾收集机制 一般使用两种方法回收垃圾; 标记清除, 在进入内存时候标记,离开内存后销毁; 引用计数; 在引用的时候计数,当不使用的时候为0;销毁; 有时候我们赋值的时候引用了全局变量,导致这个变量无法销毁,这时候我们可以手动销毁它,null; 6.性能问题; ie:window.CollectGarbage()立即执行垃圾回收; Opera7: window.opera.collect(); 7.小结 javaScript变量可以用来保存两种类型的值:基本类型值和引用类型值.基本类型的值源自以下5中基本数据类型: Undefined Null Boolean Number String 基本类型值和引用类型值具有以下特点: 基本数据类型值在内存中占据固定大小的空间,因此被保存在栈内存中; 从一个变量向另一个变量复制基本数据类型值的值,会创建一个这个值的副本; 引用数据类型值是对象,保存在堆内存中; 包含引用类型值的变量实际不是对象本身,而是一个指向该对象的指针; 从一个变量向另一个变量复制引用类型的值,复制的是指针,所以这两个变量指向同一个对象; 判断基本数据类型可以使用typeof;判断引用类型可以使用intanceof来判断是哪种对象; 所有变量都处在一个执行环境中,这个环境决定了变量的生命周,以及那一部分代码可以访问变量;以下是关于 执行环境的总结: 环境分全局环境和函数环境之分; 每次进入新的执行环境都会创建一个用于搜索变量和函数的作用域链; 函数环境不仅有权访问内部变量和函数,还可以沿作用域链访问父环境和全局环境; 全局环境只能访问到全局变量和函数,不能访问局部环境中的变量和函数; 变量的环境有助于确定何时释放内存; javaScript是一本具有垃圾回收机制的语言,开发人员不必关心内存配置问题,以下是关于垃圾回收机制的总 结: 离开作用域的值会被标记,所有在回收期间会被删除; 标记清除是现在垃圾回收机制中的主流做法,把不使用的值加上标记,然后进行回收; 引用计数方法现在已经不常用,但在ie浏览器中访问非源生javaScript对象可能会导致问题; 当代码中出现循环引用时,引用计数就会出现问题; 解除变量引用会对内存和垃圾回收有好处,所以应该在开发期间手动的解除引用;
